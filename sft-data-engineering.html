SECTION 0

全景：后训练的上限是数据给的

主综述以「算法」为轴（PPO → DPO → GRPO → Agentic RL）。本文换一根轴：把同一条流水线沿「数据」重新切开。 读完本节你会得到三个坐标系修正——流水线里多出一个 mid-training 阶段、蒸馏成为与 SFT/RL 并列的第三种原语、评测分数先问污染再谈结论。

0.1 流水线的重新切分：Pre / Mid / Post

主综述 §0 的经典流水线是 \( \text{Base} \to \text{SFT} \to \text{Preference} \to \text{RLVR} \to \text{Agentic RL} \)。 2024–2026 年的工业实践在 Base 与 SFT 之间插入了一个越来越重的阶段：mid-training（中期训练 / 退火数据工程）。 Mid-Training Survey (arXiv:2510.06826) 把它定义为预训练与任务微调之间的「退火式阶段」，沿三根轴展开： 数据分布迁移（上调 math/code/推理/指令数据）、学习率退火调度、长上下文扩展。

为什么这层不懂就解释不了 RL 结果
NVIDIA 的 Front-Loading Reasoning (arXiv:2510.03264) 证明：推理数据「前置」进预训练带来平均 +19% 的持久收益， 且后期 SFT 无法补救——even with more data。CMU 的受控实验 (arXiv:2512.07783) 进一步显示： RL 只有在预训练留下足够 headroom、且 RL 数据打在模型能力边缘时才产生真实能力增益（pass@128 提升）。 换句话说：你在 RL 阶段看到的差异，很大一部分是 mid-training 注入的能力先验决定的。 两个 RLVR 配方对比实验若底座的退火数据不同，结论可能完全不可比。

🔥 Mid-training 是正式阶段

SECTION 1

质量 vs 数量：LIMA 假说的边界与数据混合方法论

「1000 条精选样本就够了」与「3300 万条 SFT 样本」同时是 2023–2025 年的最佳实践——矛盾吗？不矛盾。 关键在于先问清楚：你要的是风格对齐还是能力注入。本章给出量级对照、配比方法论与决策依据。

1.1 LIMA 与 Superficial Alignment Hypothesis：原始表述与适用边界

先交代谱系源头。「用数据教模型听懂指令」始于 FLAN (arXiv:2109.01652, Google, 2021)： 把 60+ 个 NLP 任务用自然语言指令模板改写后做多任务微调，137B 模型在 25 个评测任务中的 20 个上 零样本超过 GPT-3 175B——首次系统证明 instruction tuning 能解锁对未见任务的零样本泛化。 Flan Collection (arXiv:2301.13688, 2023) 进一步把方法论提炼成「任务混合 + 模板多样性」： 任务配比平衡、混用 zero-shot/few-shot/CoT 模板这些「被忽视的细节」贡献了 Flan-T5 相对前作 3–17% 的提升。 这是数据工程意识在 SFT 里的第一次系统体现——§1.3 的配比方法论是它的直系后代 （系统性的谱系梳理见 Instruction Tuning Survey, arXiv:2308.10792）。 而把 SFT 的对象从「NLP 任务」换成「对话行为」、并引发质量 vs 数量之争的，是接下来的 InstructGPT 与 LIMA。

LIMA (arXiv:2305.11206, Meta) 用仅 1,000 条精心策划的 prompt-response 对 LLaMA-65B 做标准监督微调（无 RLHF）， 人评中其回答相对 GPT-4 有 43% 等同或更优、相对 DaVinci003 达 65%。论文据此提出 Superficial Alignment Hypothesis（表面对齐假说）：

模型几乎全部的知识与能力都在预训练中习得；对齐（alignment）主要是教会模型在与用户交互时该用哪种「子分布」的格式与风格。 —— LIMA: Less Is More for Alignment（意译）, arXiv:2305.11206

这个假说在 2023 年是颠覆性的，但 2024–2026 年的工业证据划清了它的适用边界：

  • 成立的范围：风格与格式对齐（对话礼仪、回答结构、拒答边界、语气）。这类「行为模式切换」确实只需要千条量级的高质量示范——Qwen3 的 Thinking Mode Fusion 用 SFT + 模板就把 think/no-think 两种行为模式融进同一模型，本质是同一原理。
  • 不成立的范围：能力注入（数学推理、代码、工具使用、多语言）。业界实际走的是「大规模合成 + 严格过滤」：Tülu 3 的 SFT 混合 939,344 条（arXiv:2411.15124），OpenThoughts3 120 万条推理样本（arXiv:2506.04178），NVIDIA Llama-Nemotron 后训练数据集 3,300 万+ SFT 样本。如果 1k 条就够，没人会花钱造 1.2M 条带消融依据的推理轨迹。
  • 更深一层：NVIDIA Front-Loading (arXiv:2510.03264) 显示能力注入甚至不该全留给 SFT——推理模式应前置到预训练/mid-training（§5.3），SFT 阶段对质量敏感（+15%）而预训练对多样性敏感（+11%）。
照搬「LIMA 1000 条达 RLHF 水平」会误导你的两种方式
① 用 1k 条数据试图教会 base model 推理/编码，然后得出「SFT 没用」的错误结论； ② 反过来无脑堆 SFT 数据量——Front-Loading 论文明确警告 naively scaling SFT data can be detrimental： 盲目扩大 SFT 量会「冲掉」早期注入的推理能力。这是很多团队「SFT 越加越差」的根因之一。 正确姿势是先回答决策树（§1.4）：要风格还是要能力。

  • 单技能上限先行：先用单技能数据训练「touchstone 模型」确认该数据池能到的上限，再谈混合——否则你分不清「混合配比不对」与「数据本身不行」。

  • 降采样优先于剔除：特大数据集（如数学合成集）整体有用但会挤占混合，先降采样观察边际效应。
  • 每轮去污染：Tülu 3 在去污染中发现多个流行开源数据集与评测集重叠（UltraFeedback 含 TruthfulQA、Evol-CodeAlpaca 含 HumanEval、NuminaMath 含 MATH 训练题——详见 §7.3）。不去污染的配比实验，分数是虚高的。

DoReMi 与 RegMix：配比变成可优化问题

预训练/mid-training 侧的配比工程，方法论由两篇工作奠定：

DoReMi (arXiv:2305.10429, NeurIPS 2023)
小代理模型 + Group DRO 学 domain 权重
用 280M 代理模型以 Group DRO（分布鲁棒优化）学出各 domain 权重，再用该权重训 8B 主模型（30× 放大）。 在 The Pile 上达到基线性能只需 1/2.6 的训练步数，few-shot 下游平均 +6.5pp——且全程不看下游任务。
RegMix (arXiv:2407.01492, ICLR 2025)
把配比当回归问题
训 512 个 1M 参数小模型（各 1B token）扫不同混合 → 拟合回归 → 外推最优配比 → 用于 1000× 大、25× 长（1B 参数 / 25B token）的模型。 以 DoReMi 约 10% 的算力取得相当效果。反直觉结论：与下游性能相关性最强的是 web 语料， 而非「看起来高质量」的 Wikipedia；domain 间交互复杂、常违反直觉，不能靠拍脑袋。
为什么 SFT 配比不能直接套 DoReMi
DoReMi/RegMix 优化的是语言建模损失的代理，对预训练/mid-training 有效；SFT 阶段目标是下游技能分数， 技能之间存在迁移与挤占（math 数据涨 code、safety 数据压 helpfulness），代理小模型的损失排序与大模型技能排序经常不一致。 这就是 Tülu 3 选择「在目标尺寸上直接迭代混合」而非代理外推的原因——贵，但所见即所得。 2026 年的 AutoSelection（下节）正在尝试把两者的优点拼起来。

1.4 从逐样本打分到配方搜索：选数据范式的 2026 转移

2023–2024 年的 SFT 数据筛选以逐样本打分为主流： IFD（Instruction-Following Difficulty, arXiv:2308.12032）用「有/无指令上下文的响应损失之比」挑「cherry 样本」， 只用 10% 原始数据即超过全量训练；DEITA (arXiv:2312.15685) 综合复杂度/质量打分器选 6K 样本， 在 MT-bench 上达 7.22（Mistral-7B 底座），与用 10× 数据的 SOTA 持平。

2026 年的 AutoSelection (arXiv:2605.12944) 指出这类方法的盲点：有效的 SFT 子集往往不是「最好的 N 条」， 而是一串有序的 curation 操作（过滤 → 混合 → 去重）共同塑造的分布。它把问题重构为 固定池数据配方搜索（fixed-pool data recipe search）：把配方的「物化」与昂贵的全量评估解耦（缓存信号）， 用高斯过程辅助排序搜索配方空间，配合 warmup probe、局部配方编辑与停滞触发重播种。 在 90K 指令池上超过全量训练、随机搜索与所有单算子基线。

决策依据：先问要什么，再选量级与方法
① 只要风格/格式（私有助手、品牌语气）→ 千条级人工精筛即可（LIMA 区间），把预算花在「写好每一条」上；
② 要注入能力（推理/代码/工具）→ 十万到百万级合成 + 严格过滤（Tülu 3 / OpenThoughts 区间），且优先考虑该能力是否应前置到 mid-training（§5）；
③ 数据池已定、想榨干 → 配方搜索（AutoSelection）优于逐样本打分（IFD/DEITA）；
④ 任何区间 → 每轮迭代都过 §7 的去污染流程，否则配比实验的反馈信号本身是脏的。
SECTION 2

SFT 是后训练里「最简单」的阶段，也是 bug 最隐蔽的阶段——因为它的失败模式不是 loss 爆炸， 而是「benchmark 正常但模型实际很差」。本章给出经验超参表与四类头号工程坑。

「超过 2 epoch 过拟合」的经验来自通用指令 SFT。在长 CoT 推理蒸馏数据上，社区多次观察到多 epoch 仍有收益 （OpenThoughts 系消融探索过 epoch 维度）。判断方法不是看 train loss，而是： ① 在未见过的推理评测上分数是否还在涨；② 采样多样性（同题多次采样的解法熵）是否塌缩。 二者背离时（分数微涨、多样性大跌）应停——多样性塌缩会直接伤害后续 RL 的探索（见 RL Pitfalls 熵塌缩一章）。

2.2 LoRA 实战：「LoRA Without Regret」的四条定量结论

三年来 LoRA 在文档/教程里通常只有一句「省显存」。Thinking Machines 的系统实验 （LoRA Without Regret, 2025-09）第一次给出了可操作的定量结论：

  1. 最优 LR ≈ 全参微调的 10 倍，且该比值近似与模型规模无关；由于 LoRA 参数化中的 \(1/r\) 缩放因子， 训练早期学习曲线与 rank 无关——最优 LR 也因此基本不随 rank 变化。直接把全参 LR × 10 是合理起点（常落在 1e-4 ~ 5e-4）。
  2. 必须挂所有层，尤其 MLP/MoE 层。只挂 attention 矩阵（早期默认做法）显著劣于只挂 MLP， 即便参数量对齐也一样；最佳实践是全部层都挂。
  3. LoRA 对大 batch 更敏感：与全参的性能差距随 batch size 增大而持续扩大（矩阵乘积参数化的优化动力学所致，非容量问题）。 预算允许时用小有效 batch（社区经验 < 32）。
  4. 「低悔恨区间」：在典型后训练数据量级、且满足前两条时，LoRA 与全参打平； 数据量超过 LoRA 容量则掉队。RL 场景更夸张：rank-1 LoRA 就能追平全参——因为策略梯度每 episode 只传递约 1 bit 信息，远低于监督学习的信息量（这个信息论视角在 §4.4 还会出现一次）。
LoRA 调参检查单
□ LR = 全参基线 × 10 起步，rank 先固定 32 不动   □ target_modules 覆盖 MLP（gate/up/down）而不只是 q/k/v/o   □ 有效 batch ≤ 32，宁可多累积步数也不放大 batch   □ 数据量 > 数百万样本或多 epoch 大语料时换全参   □ RL + LoRA 时先试小 rank（1–8），别为 rank 烧显存

2.3 Loss Masking 与 Chat Template：头号「隐形劣化」来源

SFT 的损失只应落在 assistant token 上（completion-only loss）：

Completion-only SFT Loss
$$ \mathcal{L}_{\text{SFT}}(\theta) = -\sum_{t=1}^{|y|} m_t \,\log \pi_\theta\!\left(y_t \mid x, y_{\lt t}\right), \qquad m_t = \begin{cases}1 & y_t \in \text{assistant tokens}\\[2pt] 0 & y_t \in \text{system/user/工具返回}\end{cases} $$

实现上即把非 assistant 位置的 label 置 -100。这已是主流默认（SmolLM3 等都明确采用 completion-only loss）， 但两个相邻的坑极其高发：

  • mask 偏移一个 token：chat template 的特殊 token（如 <|im_start|>assistant\n）边界算错一位， 导致模型在「回复的第一个 token」上行为微妙劣化（开头多空格、先吐一个模板碎片再说话）。症状轻微、benchmark 几乎看不出， 线上观感却明显。
  • 训练与推理模板不一致：空格、换行、system 默认值、工具调用包裹格式——任何一处不一致都会让模型在 分布外的模板上推理。这是「benchmark 正常但实际很差」的最常见原因；用同一份 tokenizer 的 apply_chat_template 同时渲染训练样本与推理请求，并 diff 检查。

多轮对话样本还有一个决策点：中间轮的 assistant 消息要不要算 loss。通用做法是都算（每轮都是监督信号）； 但对带工具调用的轨迹，工具返回（observation）必须 mask 掉——这与 agentic RL 中的 token mask 完全同构， 细节见 多轮 Tool Use 专题。

2.4 Sample Packing 的正确性：cross-contamination 坑

Packing（把多条短样本拼进一个长序列）是 SFT 吞吐优化的标配，但朴素拼接 + 不改 attention mask 是错的： causal attention 下，后一条样本的每个 token 都能看到前一条样本的全部内容——前一段对话「污染」后一段的注意力 （cross-contamination）。几乎每个训练框架历史上都踩过这个坑。正确与替代方案：

方案机制正确性吞吐采用方 / 出处
朴素 packing 直接拼接，整段 causal mask ❌ 样本间注意力泄漏 最高 历史遗留默认，应弃用
Block-diagonal mask packing 拼接 + 按样本边界切 block-diagonal mask（FlashAttention varlen / cu_seqlens） ✅ 等价于独立样本 高 当前主流正确做法；注意 position_ids 也要按样本重置
Threshold Filtering Packing 把相关样本（相似度过阈值）打进同一 pack，保留跨样本注意力当作上下文信号 ✅ 有意为之的「软污染」 高 TFP (arXiv:2408.09327, NAACL 2025)：GSM8K 最高 +7%、HumanEval +4%
Best-fit packing / grouped batching 只有当样本能完整放进同一序列才打包，绝不截断 ✅ 且无截断伤害 高（K2-V2 实测 padding < 0.004%） K2-V2 (arXiv:2512.06201)：对比 naive batching 的 >95% padding
三个连环坑，按顺序排查
① attention mask 是否 block-diagonal（varlen kernel 是否真的启用，而不是只传了参数）； ② position_ids 是否按样本重置——mask 对了但位置编码连续累加，RoPE 仍然把第二条样本当作「从位置 2048 开始」； ③ 截断策略——把一条对话从中间截成两段本身就是伤害（K2-V2 明确把 zero-truncation 当作设计目标）， 长样本宁可单独成序列。验证方法：取两条样本分别前向 vs 打包前向，逐 token 比 logits，应在数值误差内一致。

2.5 Vibe-test：benchmark 看不见的故障模式

Hugging Face 的 Smol Training Playbook （SmolLM3 3B / 11T token 训练实录）贡献了两个教科书级案例，说明为什么每个检查点都要人工对话测试（vibe-test）：

  • system prompt 被整体剥掉：数据处理管线的 bug 把训练样本里的自定义 system prompt 全部删除。 自动评测一切正常；人工一聊，发现模型完全无视 system prompt——这个故障在 benchmark 上不可见，在对话里几秒就暴露。
  • TP 组同种子 bug：tensor parallel 组内每张 GPU 用了相同随机种子，权重相关性破坏学习， 最终只能修 bug 后重启整个 11T token 的训练。

□ 人工对话 ≥ 20 轮：带/不带 system prompt、多轮、中英混合、拒答边界   □ diff 训练渲染与推理渲染的 chat template 输出   □ 抽 100 条训练样本肉眼检查（截断、空 assistant、模板碎片、思维链是否完整）  □ packing 等价性测试（§2.4）  □ 对照上一版检查点跑同一组「vibe 题」，记录主观回归   □ 评测集去污染报告随版本归档（§7）
SECTION 3

Rejection Sampling FT 谱系：从 STaR 到 agentic 冷启动

「自己采样 → 按某种信号过滤 → 在好样本上 SFT」是后训练里最朴素也最长寿的配方。 它是 Llama 系的主力、R1 冷启动的来源、agentic 轨迹合成的骨架，也是理解「SFT 与 RL 如何分工」的最佳切口。

3.1 方法源头：STaR 与 RAFT

这条谱系有两个常被遗忘的源头，理解它们就理解了之后所有变体：

STaR: Self-Taught Reasoner (arXiv:2203.14465, 2022)
自生成 → 按正确性过滤 → 再 SFT 的自提升循环
① 用 few-shot 让模型为问题生成 rationale + 答案；② 只保留答案正确的 rationale； ③ 在保留集上微调；④ 用新模型重复。点睛之笔是 rationalization：答错的题把正确答案喂给模型， 让它「倒推」出一条说得通的 rationale 再纳入训练——给困难样本兜底。 这就是后来一切 RFT / self-improvement / R1 冷启动数据的原型。
RAFT: Reward rAnked FineTuning (arXiv:2304.06767, 2023)
RM 打分版 best-of-n fine-tuning
把 STaR 的「正确性过滤」推广成「奖励排序过滤」：每个 prompt 采 n 个回答，用 reward model 选出高分样本， 丢弃低分样本，SFT，迭代。它给出了无 verifier 场景（对齐、写作）的通用形式， 也是「用 SFT 模拟 RL」路线（绕开 PPO 不稳定性）的代表。损失上等价于在 best-of-n 诱导的分布上做最大似然。
Rejection Sampling FT 的统一形式
$$ \mathcal{D}_{t+1}=\Big\{(x, y) \;\Big|\; y \sim \pi_{\theta_t}(\cdot\mid x)^{\times K},\;\; \Phi(x,y)=1 \Big\}, \qquad \theta_{t+1} = \arg\min_\theta \; \mathbb{E}_{\mathcal{D}_{t+1}}\big[-\log \pi_\theta(y\mid x)\big] $$

其中过滤器 \(\Phi\) 可以是：答案正确性（STaR / RLVR 冷启动）、RM top-1/top-p（RAFT / Llama 3）、 单测通过（SWE 轨迹）、rubric judge（Kimi K2）。从 RL 视角看，这相当于对 reward 的硬阈值加权的 policy gradient （只学正样本、权重为 1），所以它天然没有负梯度——这既是它稳定的原因，也是它上限低于 RL 的原因（无法压低坏行为的概率）。

3.2 Llama 3 的工业级流程与常见坑

The Llama 3 Herd of Models (arXiv:2407.21783) 把 rejection sampling 做成了多轮迭代后训练的主力数据来源， 其公开细节值得逐条记住：

  • 每个 prompt 采 K 个候选，K 典型取 10–30，从最新一版 chat model 采样，用 reward model 选最优；prompt 主要来自人工标注收集。
  • 后期轮次引入 system prompt 引导，让 RS 产出的回答贴合期望的语气/风格/格式（不同能力线用不同 system prompt）。
  • 工程上用 PagedAttention 提高采样吞吐——RS 的成本大头在推理，KV cache 管理直接决定可行的 K。
  • RS 产物与少量人工示范、合成数据一起进 SFT，再接 DPO，整个「RS → SFT → DPO」循环迭代多轮。
RFT 四个高频坑（生产环境实录）
① Verifier 假阳性：数学题 string-match 把「碰巧蒙对的烂推理」放进训练集；代码题弱单测放过硬编码答案。 假阳性会被 SFT 放大成系统性模式——对 verifier 的投入要按「它的错误会被蒸进模型」来定价（hack 形态详见 RL Pitfalls reward hacking 章）。
② 采样温度过低：K 个候选高度同质，过滤后等于单样本 SFT，多样性塌缩还会传染给后续 RL 的探索能力。常见做法是 T≥0.7 起步并监控候选间的去重率。
③ 不去重：同一 prompt 的多个相似正样本重复进入训练集 → 模式坍塌；prompt 级和 response 级都要去重。
④ 只保留 top-1：OpenThoughts 消融显示同题多个正样本是有效扩量手段——「每题留多个对的」常优于「每题只留最好的一条」，但前提是做了 ②③。

3.4 SFT : RL 的算力配比——不要全押任何一边

A Practitioner's Guide to Multi-turn Agentic Reinforcement Learning (arXiv:2510.01132) 用受控预算实验直接回答了配比问题。 设定：总预算 1000 个成本单位，1 条 SFT 示范的成本 = 10 个 RL episode；环境为 TextWorld（另有 ALFWorld / SWE-Gym 佐证）， 模型 Qwen 系 1.5B–8B：

  • 纯 SFT（100 条示范）：域内 95%，但复杂变体任务只有 55%——记住了套路，没学会泛化；
  • 最优配比：60 条示范 + 400 个 RL episode——基础任务 85%、复杂变体 59%，显著优于任何「全押」策略；
  • 结论：好的模仿先验大幅降低 RL 的样本需求，但 RL 不可省略——它负责把分布外泛化补上。

真实世界的同构案例：arXiv:2508.03501（长上下文多轮 SWE agent 训练）从 Qwen2.5-72B-Instruct 出发， SWE-bench Verified 基线仅 ~11%（主要败于指令跟随）；先做 RFT——每个任务跑 10 次 rollout、 只保留补丁通过测试套件的轨迹（共 6,548 条）、SFT 一个 epoch 并 mask 掉触发环境格式错误的 assistant 轮——提到 20%；再用改良 DAPO 做 RL，最终 39%。RFT 把模型「扶上正轨」，RL 把上限拉开， 两段各挣约一半，这个形状反复出现在 2025–2026 的技术报告里。

3.5 SFT 与 RL 的接口：分工的机理

SFT Memorizes, RL Generalizes (arXiv:2501.17161, ICML 2025) 用 GeneralPoints（算术卡牌）与 V-IRL（视觉导航） 的规则/视觉变体做了干净对照：SFT 倾向记忆训练分布、OOD 泛化差；结果奖励的 RL 能泛化到规则与视觉变体； 但同时——SFT 稳定输出格式是 RL 能跑起来的前提。这给了「冷启动 SFT + RL」流水线第一性的解释： SFT 负责把模型放进正确的「格式流形」，RL 负责在流形上找泛化解。

接口处的三条工程共识
① SFT 数据要贴近 base model 自身分布：直接灌别家强模型的输出（风格、用词、思维模式全是别人的）会推高初始 KL/困惑度，伤害后续 RL；缓解办法是让 base model 自己改写一遍、或干脆走 rejection sampling（数据天然 on-distribution）、 或把「SFT→RL」换成 on-policy 蒸馏这个中间档（§4）。
② 冷启动只求格式达标，不求分数：冷启动 SFT 练过头会把分布钉死，RL 阶段熵起不来。
③ 理论上 SFT 与 RL 是同一族梯度估计器的两端：标准交叉熵梯度可改写为带隐式逆概率权重 \(1/\pi_\theta\) 的 policy gradient——这个隐式 reward 病态地放大低概率示范 token，是 SFT 泛化差的一个解释； DFT (arXiv:2508.05629) 用一行代码（按 token 概率重缩放目标）修正它并提升泛化。policy gradient 的完整推导链见 derivations。
SECTION 4

知识蒸馏与 On-Policy Distillation：第三种后训练原语

2026 年的事实标准：小模型不走完整 RL 流水线，走蒸馏。On-policy distillation（OPD）以约 1/10 的 RL 成本达到同级性能， 已是 Qwen3、GLM-5、DeepSeek-V4 的共同支柱。本章从 KL 方向讲到工业配方，并给出「何时蒸馏、何时 RL」的决策表。

4.1 基础：序列级 KD、KL 方向与 exposure bias

LLM 语境下「蒸馏」一词混用了三件事，先拆开：

  • 数据蒸馏（序列级 KD / hard label）：教师生成文本，学生当普通 SFT 数据学。Alpaca、R1-Distill、OpenThoughts 都是这一类。只需要教师的文本 API。
  • Logit 蒸馏（token 级 / soft label）：学生在每个位置对齐教师的整个 next-token 分布，监督信号从每 token 1 个 hard label 变成整个词表的软分布。需要教师 logits（白盒）。
  • On-policy 蒸馏（OPD）：序列由学生采样，教师只负责逐 token 打分/给分布。这是 2025 起的关键升级。

为什么 off-policy（在教师轨迹上学）不够好？与模仿学习同样的病：exposure bias / 分布失配。 学生训练时永远走在教师铺好的「完美前缀」上，推理时却要面对自己犯错后的状态——小错累积进入它从未被纠正过的区域 （OPD Survey, arXiv:2604.00626 把这条线从模仿学习一直梳到 RLHF）。GKD 论文称之为 train-inference distribution mismatch。修法与 DAgger 同构：让学生自己走，教师在学生走过的状态上给监督。

KL 方向：mode-covering vs mode-seeking
$$ \underbrace{D_{\mathrm{KL}}\!\big(p_T \,\|\, p_S\big)}_{\text{forward KL（教师为参照）}} = \mathbb{E}_{y\sim p_T}\!\left[\log\frac{p_T}{p_S}\right] \qquad\text{vs}\qquad \underbrace{D_{\mathrm{KL}}\!\big(p_S \,\|\, p_T\big)}_{\text{reverse KL（学生为参照）}} = \mathbb{E}_{y\sim p_S}\!\left[\log\frac{p_S}{p_T}\right] $$

Forward KL 要求学生在教师所有高概率区域都给质量（mode-covering）——容量小的学生被迫「摊平」， 在教师多峰分布间插出谁都不像的混合行为。Reverse KL 只要求学生的质量都落在教师认可的区域（mode-seeking）—— 学生收敛到教师的某个模式上，行为更干净。OPD 普遍选 reverse KL，还有一个额外好处： 它以学生采样为期望分布，天然 on-policy；且「KL 低 ⇔ 教师认可」，作为奖励信号难被 hack （Thinking Machines 博客的表述：low KL always corresponds to behavior the teacher endorses）。 KL 估计器的数值细节见 derivations 的 KL 章节。

4.2 GKD：on-policy 蒸馏的奠基

Generalized Knowledge Distillation (GKD, arXiv:2306.13649, Agarwal et al., Google DeepMind) 把上面两个修正合成一个框架：

GKD 目标（简化形式）
$$ \mathcal{L}_{\text{GKD}}(\theta) = (1-\lambda)\,\mathbb{E}_{y\sim \mathcal{D}_{\text{fixed}}}\Big[\mathcal{D}\big(p_T \,\|\, p_S^\theta\big)(y)\Big] \;+\; \lambda\,\mathbb{E}_{y\sim p_S^\theta}\Big[\mathcal{D}\big(p_T \,\|\, p_S^\theta\big)(y)\Big] $$

\(\lambda\) 控制学生自采样（on-policy）的比例；散度 \(\mathcal{D}\) 可在 forward KL / reverse KL / 广义 JSD 之间选。 \(\lambda=1\) + reverse KL 就是今天通称的 on-policy distillation。GKD 还指出该框架能与 RL 微调无缝叠加。

GKD 的实验跨摘要、翻译、推理与指令微调，核心信息有二：① on-policy 数据（哪怕来自训练早期的「差学生」） 一致优于固定教师数据；② 散度选择是任务相关的超参，但 on-policy + reverse KL 在生成质量上最稳。

4.3 Qwen3 范式：off-policy 打底 + on-policy 收尾

Qwen3 Technical Report (arXiv:2505.09388) 把蒸馏正式确立为小模型生产线。 旗舰（235B-A22B / 32B）走完整四阶段：Long-CoT 冷启动 → Reasoning RL（GRPO，3,995 个 query-verifier 对，AIME'24 70.1→85.1） → Thinking Mode Fusion → 通用 RL。但 0.6B–14B dense 与 30B-A3B 全部不走这条流水线， 而是 strong-to-weak 蒸馏两阶段：

  1. Off-policy 阶段：混合教师在 /think 与 /no_think 两种模式下的输出做响应蒸馏， 让学生同时获得基础推理能力和模式切换技能——注意这一步顺带完成了 Thinking Mode Fusion 的小模型版。
  2. On-policy 阶段：学生自己采样，对齐教师（Qwen3-32B 或 235B-A22B）的 logits 最小化 KL。
两个被反复引用的数字
① 蒸馏达到显著优于 RL 的效果，只用约 1/10 的 GPU 时数（相对完整四阶段流水线）；
② logit 蒸馏不仅提 pass@1，还扩大探索空间——AIME'24/25 的 pass@64 也提升了。 这点出乎很多人意料：蒸馏没有把学生「钉死」在教师行为上，soft label 携带的「教师不确定性结构」反而比 hard label SFT 保留了更多可能性。对比之下，RLVR 常见的是 pass@1 升、pass@k 降（见主综述 §4.5 的争论）。

4.4 Thinking Machines：OPD 的机制与信息论账本

Thinking Machines Lab 的 On-Policy Distillation （Kevin Lu 等, 2025-10）把 OPD 讲成了一个简洁的公式：RL 的 on-policy 相关性 + 蒸馏的稠密监督。 流程：学生采样轨迹 → 教师对每个 token 给 log-prob → 以 per-token reverse KL 为「稠密奖励」更新学生。

Per-token Reverse KL 损失
$$ \mathcal{L}_{\text{OPD}}(\theta) = \mathbb{E}_{y \sim \pi_\theta(\cdot\mid x)} \left[\; \sum_{t} D_{\mathrm{KL}}\Big(\pi_\theta(\cdot \mid x, y_{\lt t}) \,\Big\|\, \pi_T(\cdot \mid x, y_{\lt t})\Big) \right] $$
  • 成本对照（数字引自 Qwen3 报告）：博客立论的成本表明确标注引自 Qwen3 Technical Report Table 21： 数学推理上，OPD 用 1,800 GPU 时达到 AIME'24 74.4%， 对照 RL 用 17,920 GPU 时达到 67.6%——约 1/10 成本、更高分数。 注意这组数字的出处是 Qwen3 报告本身，不是 TM 的实验。
  • 学习效率（TM 自己的独立证据）：在 self-distillation 实验中（同一 base 先用 RL 训出教师， 再用 OPD 把该策略蒸回 base），复刻一个 RL 训出来的策略，OPD 需要的梯度步数少 7–10 倍， 叠加其他因素后总算力节省可达 50–100 倍——与 Qwen3 报告的 1/10 结论在不同实验设置下互证同一量级。
  • 信息论账本：RL 的结果奖励每 episode 只透出 O(1) bit（成/败）， 而蒸馏每个 token 都有监督，每 episode 透出 O(N) bit。这就是「蒸馏比 RL 快得多」的第一性解释—— 也呼应 §2.2 里「RL 信息量低到 rank-1 LoRA 就够」的观察：两者是同一枚硬币的两面。
  • 持续学习场景：在新领域上做 SFT 后，模型常灾难性遗忘对话/指令行为；用「老的自己」当教师做 OPD 能把行为恢复回来，且不抹掉新知识——「学新知识」与「恢复行为」可以交替推进。这为内部助手/个性化模型给出了 不依赖 RL 的更新循环。
OPD 不是免费午餐
① 它的上限是教师——教师没有的能力蒸不出来，「探索发现新解法」仍是 RL 独有的； ② 教师必须能对学生 token 给出有意义的 log-prob：师生 tokenizer 不同时需要黑盒方案（§4.5）； ③ THUNLP 的机制分析（§4.6）警告 dense token 级监督的「免费午餐」在长序列上有代价，且师生思维模式不兼容时 OPD 白做。

4.5 黑盒蒸馏：GAD 把判别器当 on-policy 奖励

如果教师是闭源 API（只有文本输出、没有 logits）呢？Black-Box On-Policy Distillation (arXiv:2511.10643, Microsoft Research) 提出 GAD（Generative Adversarial Distillation）：学生当 generator， 另训一个 discriminator 区分「学生回答 vs 教师回答」。判别器实质上是一个与学生共同演化的 on-policy reward model， 提供稳定、自适应的反馈。结果：Qwen2.5-14B-Instruct 学生在 LMSYS-Chat 自动评测上达到教师 GPT-5-Chat 同级， 全面优于序列级 KD。

GAD 的意义在于补全了矩阵的最后一格：on-policy × 黑盒。四象限对照：

Off-policy（教师轨迹）On-policy（学生轨迹）
白盒（有 logits） 经典 logit KD（soft label SFT） GKD / Qwen3 第二阶段 / TM OPD（reverse KL）
黑盒（仅文本） 序列级 KD：Alpaca、R1-Distill、OpenThoughts GAD（判别器当 on-policy RM）

4.6 成败条件与教师选择：三个反直觉结论

① 教师不是越强越好（capacity gap 定律）

Distillation Scaling Laws (arXiv:2502.08606, Busbridge et al., Apple) 用大规模实验拟合出蒸馏的 scaling law： 学生性能由教师交叉熵、学生规模、蒸馏 token 数共同决定，并确认了 capacity gap 现象—— 更强的教师可能教出更差的学生，且该差距源于师生「学习能力」（假设空间 + 优化能力）的差距， 不只是参数量之差。实操含义：最优教师只需比学生强一档；以及一个冷静的提醒—— 若你只打算训一个学生且教师也要现训，监督学习通常优于蒸馏； 蒸馏的优势区在「教师已存在」或「一个教师喂多个学生」（恰好就是工业模型家族的形态）。

② 中档教师赢过旗舰教师的实证

OpenThoughts (arXiv:2506.04178) 的 1000+ 消融给出最常被引用的例子：QwQ-32B 当教师， 学生成绩一致优于用更强的 DeepSeek-R1 当教师——尽管 QwQ-32B 自己在目标 benchmark 上分数更低。 OpenThinker3-7B（QwQ 教师 + 1.2M 数据）对 R1-Distill-Qwen-7B 的提升：AIME 2025 +15.3pp、 LiveCodeBench +17.2pp、GPQA-D +20.5pp。另一个同方向结论：各种 answer 端的校验/过滤花活收益甚微， 不如把对的样本都留下（数量本身就是信号）。

③ OPD 的两个生效前提（THUNLP）

Rethinking On-Policy Distillation (arXiv:2604.13016, THUNLP) 给出 OPD 成败的判据： (a) 师生思维模式（thinking pattern）兼容；(b) 教师确实拥有学生训练分布之外的「真正新能力」—— 两者缺一，OPD 白做。机制上，有效的 OPD 表现为「学生访问状态上的高概率 token 向教师渐进对齐」， 且存在一个集中了 97%–99% 概率质量的小型师生共享 token 集。对失败场景论文给出两个补救： off-policy 冷启动（先把思维模式拉兼容）与 teacher-aligned prompt 选择。 这解释了为什么 Qwen3 一定要先跑 off-policy 阶段再上 on-policy——顺序不是随意的。

4.7 工业落地与「蒸馏 vs RL」决策

2026 年的旗舰报告里，OPD 已经出现在流水线的三个不同位置：

  • 量产小模型（Qwen3）：strong-to-weak 两阶段，替代整条四阶段流水线；
  • 跨阶段防遗忘（GLM-5, arXiv:2602.15763）：Reasoning RL → Agentic RL → General RL 顺序流水线中， 用 On-Policy Cross-Stage Distillation 在阶段间传递能力、防止后一阶段冲掉前一阶段；
  • 专家合并（DeepSeek-V4 报告， 模型卡附 PDF）：先分别养成 10+ 个领域 specialist（数学/代码/agent/指令等，各自 SFT + GRPO RL）， 再在学生 on-policy rollout 上对多教师分布做加权 reverse-KL 蒸馏，把多个专家的能力合并进单一模型 （V4-Pro 1.6T-A49B）——以蒸馏替代难调的多域混合 RL。

SECTION 5

Mid-training 与退火数据工程：后训练的上游决定下游

「推理能力应该在 LR 退火期注入」已从技巧升格为共识；mid-training 也从临时叫法变成有 survey、有分类法的正式阶段。 这一章解释机制、给出各家打法对照，并讲清 2025 年底出现的新争议：好数据放最后，会撞上最低的学习率。

5.1 WSD 调度与 MiniCPM 的发现

故事从学习率调度开始。MiniCPM (arXiv:2404.06395) 推广的 WSD（Warmup-Stable-Decay）调度把训练分成三段：

WSD Learning Rate Schedule
$$ \eta(t)=\begin{cases} \dfrac{t}{T_w}\,\eta_{\max} & t \le T_w \quad\text{(warmup)}\\[6pt] \eta_{\max} & T_w < t \le T_s \quad\text{(stable：恒定高 LR，可无限续训)}\\[6pt] \eta_{\max}\cdot f\!\left(t-T_s\right) & t > T_s \quad\text{(decay：快速退火，}f\text{ 取指数/线性衰减)} \end{cases} $$

与 cosine 不同，WSD 的 stable 段不预设总步数——随时可以从 stable 段分叉出一次 decay 得到「收敛版」模型， 天然支持持续训练与数据消融（MiniCPM 用它低成本研究 data-model scaling law）。decay 段是损失骤降的窗口。

MiniCPM 的关键实验发现：在 decay 段混入高质量 SFT 级数据（与预训练数据混合）， 收益远高于把同样数据放到 SFT 阶段再加——论文的结论是「模型的专门化与能力增强应当从 decay 阶段开始」。 机制直觉：decay 段 LR 仍然「够高」、模型可塑性还在，但分布已接近收敛、不会把高质量数据的信号冲散； 而到了 SFT 阶段 LR 低两个数量级、能写入的东西有限。

Llama 3 首创公开的「annealing 评估法」——小数据集质量的标准量具
想知道一个候选数据集有没有价值？Llama 3 的做法：取一个训练到 50% 的 8B 检查点， 把 LR 线性退火到 0、跑 40B token，期间给候选数据集设定上采样权重，看 benchmark 变化。 这比为每个小数据集跑 scaling law 实验便宜得多，已成为业界评估退火数据的标准方法。 注意它评估的是「在退火窗口的边际价值」——与 RegMix 评估「全程配比」回答的是不同问题。
术语警告：各家的「mid-training」不是同一个东西
有的报告指「退火段换数据」（OLMo 2 Dolmino），有的指独立的第二阶段（Olmo 3 的 100B Dolmino 阶段）， 有的把长上下文扩展也算进来（Longmino / Mid-Training Survey 的第三轴），还有的把 agentic 持续预训练（CPT）也叫 mid-training。 读技术报告时不要假设同名同义，先查它的 LR 调度与数据切换点。

5.3 推理数据前置：Front-Loading 与 RL 的相互作用

NVIDIA 的 Front-Loading Reasoning (arXiv:2510.03264) 用受控实验回答「推理数据放哪一阶段」：

  • 推理数据前置到预训练带来平均 +19% 的持久收益，且这部分能力「奠基」之后， 后期 SFT 用再多数据也补不回来；
  • 最优分配是非对称的：预训练吃多样性（推理模式的广度，+11%），SFT 吃质量（+15%）；
  • 高质量预训练数据有潜伏效应——收益要等 SFT 之后才显现（只评 base model 会低估它）；
  • 盲目扩大 SFT 数据量有害，会冲掉早期注入的推理能力。

把视角再拉到 RL：On the Interplay of Pre-Training, Mid-Training, and RL (arXiv:2512.07783) 的受控结论是 RL 只有在预训练留下足够 headroom、且 RL 数据打在能力边缘（edge of competence）时才产生 pass@128 级的真实增益； 固定算力预算下「加 mid-training」显著优于「全给 RL」。这两篇合起来，给了 pass@k 争论（主综述 §4.5）一个务实的落点： 与其争论 RL 能否扩展能力边界，不如先检查你的 mid-training 有没有把先验铺够。

闭环的最后一块：ReMiT (arXiv:2602.03075) 把方向反过来——用 RL 调优后的模型给 mid-training 的 token 动态重加权 （识别对推理重要的 token），让「RL 强化的能力」回流到 base model，10 个预训练 benchmark 平均 +3%， 且收益在后训练各阶段保持 +2% 以上。pre→mid→post 的单行道正在变成循环。

5.4 新争议：LR × 课程的交互陷阱

「好数据放退火段」的朴素推论是「数据按质量升序排课程」。 How Learning Rate Decay Wastes Your Best Data (arXiv:2511.18903) 指出这撞上了一个结构性矛盾： 质量升序课程把最好的数据排在训练末尾，而那正是 LR 最低、有效学习量最小的时刻。 论文的对策：用更温和的 LR 衰减 + 模型检查点平均（checkpoint averaging）恢复课程收益， 无需任何额外数据处理即获得平均 +1.64% 的 benchmark 提升。

怎么调和「MiniCPM 说放 decay 段好」与「2511.18903 说 decay 浪费好数据」？
两者对照的基线不同。MiniCPM 比较的是「decay 段注入 vs SFT 段注入」——decay 赢； 2511.18903 比较的是「decay 末端注入 vs 课程与 LR 联合设计」——联合设计赢。 合并后的工程结论：高质量数据进 decay 段没错，但 ① decay 不要太陡（温和衰减）， ② 最好的数据不要堆在最后一刻，可提前到 decay 前中段或多次复访，③ 用 checkpoint averaging 把 decay 段的多个状态合起来。 课程顺序必须与 LR 调度联合设计，单独优化任何一个都会留钱在桌上。

5.5 实操总结

  • 预算分配口诀：能力先验（推理/代码/领域知识）尽量前置——预训练求多样、mid-training 求质、SFT 求格式与精、RL 求边缘突破。每往后挪一个阶段，同一份数据的「写入深度」下降一档。
  • 候选数据先过 annealing run（Llama 3 方法）再决定是否买/造全量。
  • 解释 RL 实验差异时，先对齐底座的 mid-training 配方；跨底座结论（尤其只在 Qwen 系上验证的）要打折——Qwen 预训练里已含大量可被 RL「激发」的推理行为，这正是 mid-training 先验的体现。
  • 退火数据本身要去污染：decay 段的数据「写入深度」最大，一旦混入评测集变体，污染也最深（§7）。
SECTION 6

现代合成数据管线：从 Self-Instruct 到闭环造数

主综述对合成数据的覆盖止于 2023 年的 Self-Instruct / Evol-Instruct。此后三年，合成数据从「便宜替代品」 变成了 Phi 系的主菜、Nemotron 的开放资产、以及 2026 年可被 RL 直接优化的「策略」。

6.1 谱系：四代方法一张图

2022-12 · 第一代：自举
Self-Instruct (arXiv:2212.10560)
用模型自己生成指令-输入-输出三元组，过滤无效/重复后微调自身；vanilla GPT-3 在 Super-NaturalInstructions 上 +33%，逼近 InstructGPT-001。Alpaca（175 seed → 52K, davinci-003 蒸馏）把它变成开源运动。
2023-04 · 第二代：演化
Evol-Instruct / WizardLM (arXiv:2304.12244)
对种子指令做 in-depth 演化（加约束、深化、增加推理步数）与 in-breadth 演化（扩话题/技能面），解决 Self-Instruct 产物复杂度不足的问题。UltraChat、Vicuna 等同期沿对话线扩量。
2023-06 → 2024-12 · 第三代：体系化（质量工程）
Phi 系「教科书路线」与 Magpie、Nemotron
合成数据从「佐料」变「主菜」：管线类型化（rewrite / self-revision / instruction reversal…）、以可控质量大规模量产，并出现零种子提取（Magpie）与开放数据集资产化（Nemotron）。详见 §6.2–6.4。
2026 · 第四代：闭环优化
DPG：数据生成器本身成为 RL 策略 (arXiv:2604.08423)
不再人工设计管线，而是以「下游模型在目标指标上的表现」为奖励直接优化数据生成器——合成数据从静态管线走向闭环（§6.5）。

6.2 Phi 系：textbook 质量路线的两个里程碑

phi-1 · Textbooks Are All You Need (arXiv:2306.11644)
1.3B 参数 · 4 天 × 8×A100
只用 7B token——6B 经过「教科书质量」过滤的 web 代码 + 1B GPT-3.5 合成的教科书与习题—— 就在 HumanEval 拿到 50.6% pass@1、MBPP 55.5%， 350M 的 phi-1-small 也有 45%。证明了「质量过滤 + 定向合成」能让小模型越级。
Phi-4 Technical Report (arXiv:2412.08905)
14B 参数 · 合成数据为主菜
50 大类合成数据集，各自依赖不同种子与多阶段 prompting，总计约 400B unweighted token。 管线类型包括：multi-agent prompting（多模型出解互滤）、self-revision（模型按 rubric 自我批改迭代）、 instruction reversal（从代码反推指令，保证指令-代码对的紧致性）。 结果：在 STEM 问答上超过教师 GPT-4——蒸馏数据若组织得好，学生可以越过教师（呼应 §4.6 的教师选择结论）。
Phi 路线的方法论本质
不是「用大模型生成数据」这件事新，而是把每条管线当产品管理：种子来源、生成程序、过滤器、混合权重全部显式化、可消融。 50 类管线意味着 50 个可独立调优的质量旋钮——这与 §1.3 的「配比是科学」一脉相承。 风险同样明确：合成分布会复制教师的盲区与污染（教师背过的评测题会变成「合成原创题」），这是 §7.1 蒸馏污染传导的入口。

6.3 Magpie：对齐数据可以被「反向提取」

Magpie (arXiv:2406.08464, ICLR 2025) 的发现简洁得近乎魔术：对齐过的模型（如 Llama-3-Instruct）拿到 只有 chat template 左半边（到 user 消息位置为止）的输入时，会自己「吐出」一条像模像样的用户指令—— 因为这就是它对齐训练分布的自回归延续。流程：

Magpie：零种子合成对齐数据
self-synthesis
for i in range(N):
  query    = LLM_aligned("<|begin|><|user|>")   # 只喂模板前缀 → 模型自吐指令
  response = LLM_aligned(template(query))  # 正常生成回答
# 4M 对 → 质量/难度/多样性打分 → 精选 300K → SFT base model

用这 300K 数据微调 Llama-3-8B-base，效果与官方 Llama-3-8B-Instruct 相当——而官方版用了 1000 万量级的 SFT 数据 + 后续偏好优化。两个推论：① 对齐模型的指令分布是可以被零成本「抽取」的资产 （也是安全/合规上的提示）；② 指令分布的覆盖面可能比人工设计的种子更接近真实用户分布—— 这是它打平官方配方的原因之一。

6.4 Nemotron：合成数据的开放资产化

NVIDIA 把合成数据当成可发布的基础设施在运营： Llama-Nemotron Post-Training Dataset 公开了 3,300 万+ SFT 样本（code/math/science/chat/safety） 与 5.6 万条 RL 指令数据；Nemotron-Post-Training-Dataset-v2 在此之上把 SFT/RL 数据扩展到五种语言 （西/法/德/意/日），合成教师包括 DeepSeek-R1-0528 与 Qwen3-235B-A22B 等。 配合 §5.3 的 Nemotron-Pretraining-SFT 数据（前置推理路线），NVIDIA 路线的特征是： 教师明示、管线明示、可商用——把「数据壁垒」翻译成「数据资产」。 对从业者的直接价值：这是少数能直接拿来复现「能力注入量级」（§1.2）的全开放语料。

6.5 DPG：把数据生成器当 RL 策略

Synthetic Data for any Differentiable Target (arXiv:2604.08423) 提出 Dataset Policy Gradient： 把「生成一批训练样本」视为动作，把「目标模型在这批样本上 SFT 后、在任意可微目标上的表现」视为奖励， 用高阶梯度做数据归因（attribution），归因分数作为 policy reward 更新生成器。

DPG 的目标结构（示意）
$$ \max_{\phi}\;\; \mathbb{E}_{\mathcal{D}\sim G_\phi}\Big[\, J\big(\,\theta^\ast(\mathcal{D})\,\big)\Big], \qquad \theta^\ast(\mathcal{D}) = \mathrm{SFT}(\theta_0;\, \mathcal{D}) $$

生成器 \(G_\phi\) 的梯度穿过「学生被这批数据训练之后的表现」\(J\)。论文用「只靠 SFT 合成样本就把 QR 码图案写进模型权重」 这类演示证明了控制的精度。意义：§1 的「配方搜索」还在离散空间里挑选，DPG 直接对生成分布做连续优化—— 「为目标能力定向造数」第一次成为端到端可优化的问题。

合成数据的三个系统性风险
① 分布塌缩/模式单一：合成数据反复喂回训练会收窄分布尾部——管线里必须有多样性度量与去重（§3.2 的坑在更大尺度重演）； ② 蒸馏污染传导：教师背过的评测集会以「原创面目」混进合成数据（§7.1），对 AIME/GPQA 这类小评测集是致命的； ③ 偏好与事实偏差放大：教师的系统性偏好（格式、立场、语言习惯）会在学生身上被放大—— 多教师混合（如 Nemotron v2 同时用 R1 与 Qwen 系）是廉价的对冲。
SECTION 7

数据去污染与基准污染审计：先问污染，再谈结论

没有去污染流程的数据工程是在给自己下毒：你以为在优化能力，其实在优化背题。 本章覆盖训练侧防线、评测侧审计、蒸馏时代的隐蔽传导，以及 2025 年出现的新形态——agent 当场搜到答案。

7.1 污染类型学：四条进入路径

① 直接污染（训练集含原题）
评测题原文或近似变体进入预训练/SFT 语料。最经典的形态；n-gram 防线主要防它。
② 改写污染（paraphrase / 翻译 / 格式变换）
同一道题换说法、换语言、换数字模板。n-gram 完全失效，需要 embedding / LLM 模糊匹配； 2026 年的立场文进一步指出连「混淆格式」模型都见过，主张 benchmark 本身要设计成抗污染（动态/再生成式）。
③ 蒸馏传导污染（合成数据时代的主形态）
教师背过评测集 → 教师生成的「原创」训练数据携带评测题的解法/原题变体 → 学生隐蔽污染。 你对自己的语料做了完美去污染也防不住它，因为污染在上游教师的权重里。对 AIME（每年 30 题）、GPQA 这类小评测集致命。 对策：对合成数据本身再过一遍去污染；优先用日期切割评测（§7.3）验证。
④ Search-time 污染（agent 时代新形态）
带搜索的 agent 在评测当下直接检索到基准数据集与标准答案（详见 §7.4）。 训练侧零污染也挡不住——污染发生在推理时。

7.2 训练侧三层防线与各家口径

训练侧去污染的标准结构是三层叠加，逐层换精度换召回：

  1. 第一层：exact / hash 匹配——全串或规范化（去空白/符号/小写）后哈希查重。零误杀，但只能抓完全复制。
  2. 第二层：n-gram overlap——业界主力，n 取 8–13。各家口径（均已核实出处）： GPT-2 用 8-gram 做事后分析，GPT-3 按 13-gram 碰撞扫描，GPT-4 改为每条评测样本抽 3 个随机 50 字符子串做碰撞（去空格符号后），Tülu 3 用 8-gram（细则见下）。n 越小越严格、误杀越多—— 报告分数时必须同时报告去污染口径，不同口径的分数不可比。
  3. 第三层：embedding 相似度 + LLM 模糊匹配——抓改写/翻译变体。注意 Tülu 3 的实测教训： embedding 法难以区分「真改写」与「只是分布相似」，他们最终以 n-gram 为主力；LLM judge 逐对判定精度最高但只适合小规模复核。
Tülu 3 协议：可以直接抄的去污染规范 (arXiv:2411.15124)
token 级 8-gram 匹配：测试样本中某 token 若与某训练样本共享一个含它的 8-gram 即记为命中； 一条测试样本若 >50% 的 token 命中同一条训练样本 → 判定该对实例重叠； 一个训练集若与任一评测集的 >2% 实例重叠 → 判定该训练集污染、整体处理。 用此协议他们发现多个流行开源数据集中招：UltraFeedback ↔ TruthfulQA、Evol-CodeAlpaca ↔ HumanEval、 NuminaMath ↔ MATH、WildChat ↔ 安全评测。教训：开源数据集 ≠ 干净数据集， 去污染后部分榜单分数回落是正常且正确的——那部分分数本来就是虚高的。

7.4 Search-time contamination：评测时的现行犯

Scale AI 的 Search-Time Data Contamination (arXiv:2508.13180) 审计了带搜索能力的 agent（Perplexity 的 Sonar Pro / Sonar Reasoning Pro / Sonar Deep Research）：在 HLE、SimpleQA、GPQA 三个基准上， 约 3% 的题目中 agent 直接在 HuggingFace 上检索到了带 ground-truth 标签的数据集—— 有的还在推理链里坦然写明「我在 HF 上找到了这组问答对」。把 HuggingFace 从可访问源里禁掉后， 这部分题目的准确率下降约 15%。

  • 对评测方：agentic 评测必须封网或对检索结果做基准过滤（域名黑名单 + 答案串匹配）， 并把「检索到题库」记为无效样本；评测报告应披露网络策略。
  • 对训练方：RL 训练环里若有真实搜索工具，reward 可能被「搜到答案」捷径骗过—— 这是 reward hacking 的数据侧变体（hack 形态总表见 RL Pitfalls）； SWE 任务同理：模型可能背过 GitHub 上的修复 commit，SWE-bench 横向比分要警惕记忆污染。

8.3 十条带出处的「硬」结论

  1. 风格对齐千条够，能力注入按十万–百万级规划——且先考虑能否前置（LIMA；Tülu 3；Front-Loading）。
  2. 盲目堆 SFT 数据会冲掉早期推理能力——SFT 不是越多越好（arXiv:2510.03264）。
  3. SFT 全参起点：lr 5e-6（8B）/2e-6（70B）、linear decay、batch 128、2 epochs（Tülu 3 搜索结论）。
  4. LoRA：LR×10、挂 MLP、全层、小 batch；后训练量级内与全参打平，RL 下 rank-1 即可（LoRA Without Regret）。
  5. packing 必须 block-diagonal mask + 重置 position_ids，或用 best-fit 零截断方案（TFP；K2-V2）。
  6. RFT 全押或 RL 全押都次优——60 示范 + 400 episodes 型的混合配比最优（arXiv:2510.01132；2508.03501 的 11→20→39）。
  7. 小模型走蒸馏不走 RL：~1/10 成本、pass@k 不塌（Qwen3；TM OPD；R1 蒸馏 72.6% vs 直接 RL ~47%）。
  8. 教师不是越强越好：capacity gap 是定律级现象（Distillation Scaling Laws；OpenThoughts 的 QwQ>R1）。
  9. 高质量数据进退火段，但课程要与 LR 联合设计（MiniCPM；arXiv:2511.18903）。
  10. 分数先过污染审计：私有 holdout 上部分模型族掉分（GSM1k）；agent 会当场搜到答案（STC ~3% 题目）。
